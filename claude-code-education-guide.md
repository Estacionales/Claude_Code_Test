# Claude Code 팀 교육 과정 — 수강 가이드

> 작성일: 2026-06-30
> 대상: 팀 단위 Claude Code 활용 교육 수강자

---

## 개요

본 교육은 Claude Code를 단순 코드 자동완성 도구가 아닌 **엔지니어링 시스템의 구성 요소**로 활용하는 것을 목표로 합니다. 5개 파트로 구성되며, 각 파트는 실습 중심으로 진행됩니다.

---

## 파트별 학습 내용 및 사전 준비

### Part 1. 하네스 엔지니어링

#### 무엇을 배우나?

| 주제                    | 핵심 내용                                                                   |
| ----------------------- | --------------------------------------------------------------------------- |
| Agent = Model + Harness | AI 모델 자체와 그것을 감싸는 실행 환경(Harness)이 어떻게 분리되는지 이해    |
| CLAUDE.md               | 프로젝트 루트에 두는 AI 행동 지침 파일 작성법 (컨텍스트 주입, 규칙 선언)    |
| SKILL.md / Skill 파일   | 반복 작업을 슬래시 커맨드로 캡슐화하는 방법                                 |
| 서브에이전트            | 작업을 병렬 Agent로 위임하는 패턴 (Explore, Plan, Code-review 등)           |
| Hooks                   | 도구 호출 전후에 자동 실행되는 셸 스크립트 설계                             |
| 권한(Permissions)       | allowedTools / deniedTools 로 AI의 행동 범위를 제한하는 방법                |
| MCP                     | Model Context Protocol — 외부 시스템(DB, API, 파일 시스템)을 AI 도구로 연결 |

#### 실습에서 실제로 하는 것

- `CLAUDE.md` 파일을 직접 작성하여 프로젝트 규칙을 AI에 주입
- Hook 스크립트로 "파일 저장 시 lint 자동 실행" 등 자동화 파이프라인 구성
- MCP 서버를 연결하여 Claude Code가 외부 데이터를 조회하도록 설정
- 서브에이전트에게 태스크를 위임하고 결과를 통합하는 오케스트레이터 패턴 구현

#### 사전에 알아두면 좋은 것

```
필수
- CLI(터미널) 기본 사용법 (경로 이동, 파일 생성, 환경 변수)
- JSON 문법 (settings.json 직접 편집)
- Git 기초 (add / commit / branch 정도)

권장
- 셸 스크립트(bash/PowerShell) 기초 — Hook 작성에 사용
- IDE 플러그인 개념 이해 (VS Code Extension 수준)
- REST API 개념 (MCP 연동 실습 시 필요)
```

### Part 2. SDD (Spec-Driven Development)

#### 무엇을 배우나?

SDD는 **"검증 가능한 명세를 먼저 작성하고, AI가 그 명세를 코드로 구현"** 하게 하는 개발 방법론입니다.

| 단계  | 폴더            | 하는 일                        |
| ----- | --------------- | ------------------------------ |
| 1단계 | `sdd/planning/` | 요구사항 수집, 이해관계자 정의 |
| 2단계 | `sdd/plan/`     | EARS 문법으로 명세 작성        |
| 3단계 | `sdd/build/`    | 명세 기반 코드 생성            |
| 4단계 | `sdd/verify/`   | 명세 대비 자동 검증 (테스트)   |
| 5단계 | `sdd/operate/`  | 운영 모니터링, 피드백 루프     |

**EARS 문법 예시:**

```
[WHEN 사용자가 로그인 버튼을 누르면]
[WHILE 이메일 형식이 유효하지 않으면]
[THE SYSTEM SHALL 오류 메시지를 표시해야 한다]
```

#### 실습에서 실제로 하는 것

- EARS 문법으로 기능 명세서 직접 작성
- 작성한 명세를 Claude Code에 입력하여 코드 골격 자동 생성
- 명세에서 테스트 케이스를 자동 도출하는 워크플로 실습
- 명세 변경 → 코드 재생성 → 검증 사이클 반복 경험

#### 사전에 알아두면 좋은 것

```
필수
- 요구사항 문서(기획서, 기능 명세) 읽고 쓰는 경험
- 조건문/제어 흐름 이해 (if/when/while 로직)

권장
- 유스케이스(Use Case) 또는 사용자 스토리(User Story) 작성 경험
- BDD(Behavior-Driven Development) 개념 (Given-When-Then)
- 단위 테스트 작성 경험 (어느 언어든 무관)
```

### Part 3. 실전 예제

#### 무엇을 배우나?

두 가지 규모의 실전 시나리오를 통해 앞선 개념을 종합 적용합니다.

**예제 A: 이커머스 DDD (도메인 주도 설계)**

```
사용 기술: Java, JUnit, MockMvc
학습 포인트:
- 도메인 모델(Entity, Aggregate, Repository) 설계를 AI와 함께 진행
- MockMvc 기반 E2E 테스트를 명세에서 자동 생성
- 레이어 간 의존성 규칙을 CLAUDE.md에 선언하여 AI가 위반하지 않도록 강제
```

**예제 B: 대규모 MSA 병렬 에이전트**

```
학습 포인트:
- 여러 마이크로서비스를 병렬 서브에이전트로 동시 개발
- 서비스 간 계약(API Contract)을 명세로 선언
- 팀원 각자가 다른 서비스 에이전트를 담당하는 협업 시뮬레이션
```

#### 실습에서 실제로 하는 것

- DDD 패턴대로 폴더/클래스 구조를 AI가 자동 생성하도록 프롬프트 설계
- JUnit + MockMvc 테스트 파일을 명세에서 일괄 생성
- MSA 환경에서 서비스 A 팀과 서비스 B 팀이 병렬로 개발 후 통합

#### 사전에 알아두면 좋은 것

```
필수
- Java 또는 팀 주력 언어 기초 (클래스, 인터페이스 개념)
- REST API 설계 기초 (HTTP 메서드, 상태 코드)

권장
- DDD 기본 용어 (Entity, Value Object, Aggregate, Repository)
- JUnit 테스트 작성 경험
- Docker Compose 기초 (MSA 로컬 실행)
- Spring Boot 프로젝트 구조 이해
```

### Part 4. 레거시 전환 전략

#### 무엇을 배우나?

**Strangler Fig 패턴**을 활용해 레거시 시스템을 무중단으로 현대화하는 전략을 Claude Code와 함께 실습합니다.

```
Strangler 패턴 흐름:
┌─────────────────────────────┐
│  레거시 시스템               │
│  ┌─────────┐                │
│  │ 모듈 A  │ ← 신규 코드로 교체
│  │ 모듈 B  │ ← 유지         │
│  │ 모듈 C  │ ← 다음 단계    │
│  └─────────┘                │
└─────────────────────────────┘
```

| 단계 | 내용                                   |
| ---- | -------------------------------------- |
| 분석 | 레거시 코드 의존성 맵 자동 생성        |
| 격리 | 교체 대상 모듈 경계(Facade) 설정       |
| 이식 | SDD 명세 기반으로 신규 모듈 재작성     |
| 검증 | 구버전 / 신버전 동시 실행 후 결과 비교 |
| 절체 | 트래픽을 점진적으로 신규 모듈로 이동   |

#### 실습에서 실제로 하는 것

- 주어진 레거시 코드를 Claude Code로 분석, 의존성 다이어그램 생성
- 교체 우선순위 모듈을 SDD 명세로 재정의
- 기존 테스트를 신규 모듈에 재사용하는 회귀 테스트 전략 수립

#### 사전에 알아두면 좋은 것

```
필수
- 레거시 코드가 무엇인지 경험적으로 알고 있을 것 (리팩터링 경험)
- 인터페이스/추상화 개념 이해

권장
- 디자인 패턴 기초 (Facade, Adapter, Proxy)
- 의존성 주입(DI) 개념
- 통합 테스트 vs 단위 테스트 차이 이해
```

### Part 5. AI 거버넌스

#### 무엇을 배우나?

**AI 기본법 4대 의무**와 이를 SDD 기반으로 자동 증빙하는 방법을 학습합니다.

| 의무        | 내용              | SDD 대응 방법                                        |
| ----------- | ----------------- | ---------------------------------------------------- |
| 투명성      | AI 활용 사실 고지 | CLAUDE.md에 AI 사용 범위 명시, 커밋 메시지 자동 태깅 |
| 설명 가능성 | AI 결정 근거 제시 | 명세(EARS) → 코드 추적 가능성 확보                   |
| 안전성      | 위험 통제         | Hook으로 위험 명령 차단, 권한 최소화                 |
| 책임성      | 인간 최종 결정    | 승인 게이트(Approval Gate) 설계                      |

#### 실습에서 실제로 하는 것

- AI가 생성한 코드에 자동으로 출처 주석을 다는 Hook 작성
- SDD 명세 파일을 감사 증빙으로 패키징하는 스크립트 구성
- 고위험 작업(파일 삭제, 배포)에 인간 승인을 강제하는 권한 설계
- 거버넌스 체크리스트를 Claude Code 워크플로에 통합

#### 사전에 알아두면 좋은 것

```
필수
- AI 도구가 생성한 산출물의 법적/윤리적 책임 소재에 대한 기본 인식

권장
- 사내 AI 활용 정책 또는 가이드라인 숙지
- 감사 로그(Audit Log) 개념
- CI/CD 파이프라인에서 승인 단계(Approval Gate) 경험
```

## 팀 공통 사전 준비 체크리스트

교육 당일 원활한 진행을 위해 팀 전체가 아래를 미리 준비해 주세요.

### 환경 설정

- [ ] Claude Code CLI 설치 완료 (`claude --version` 확인)
- [ ] Anthropic API Key 발급 및 환경 변수 설정
- [ ] VS Code 또는 JetBrains IDE에 Claude Code 확장 설치
- [ ] Node.js 18+ 또는 Java 17+ 설치 (실습 예제에 따라)
- [ ] Git 설치 및 계정 설정

### 개념 사전 학습 (30분 투자 권장)

1. **Claude Code 공식 문서** 중 "CLAUDE.md 작성법" 섹션 일독
2. **EARS 문법** 요약 1페이지 읽기 (검색: "EARS requirements notation")
3. **Strangler Fig 패턴** 마틴 파울러 블로그 포스트 일독

### 팀 협의 사항

- [ ] 실습용 공통 GitHub 조직(Org) 또는 저장소 생성
- [ ] 팀원별 역할 분담 사전 협의 (MSA 실습 시 서비스 분담)
- [ ] 사내 AI 활용 정책 담당자 사전 확인 (거버넌스 파트 대비)
